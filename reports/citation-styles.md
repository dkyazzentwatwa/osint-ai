# Citation Styles v2.0

Academic citation guidelines for OSINT investigations.

---

## 1. APA Format Guide (7th Edition)

### In-Text Citations

**Basic Format:**
```
(Author, Year)              # e.g., (Smith, 2024)
(Author, Year, p. X)        # e.g., (Smith, 2024, p. 15)
Author (Year) stated...     # e.g., Smith (2024) stated...
```

**Multiple Authors:**
```
(Smith & Jones, 2024)       # 2 authors
(Smith et al., 2024)      # 3+ authors
(Smith, 2024; Jones, 2023) # Multiple citations
```

**No Author (Use title):**
```
("Understanding Cyber Threats," 2024)
(Webpage Title, n.d.)     # No date
```

**Organization as Author:**
```
(Federal Bureau of Investigation, 2024)
(FBI, 2024)               # Acronym after first use
```

### Reference List Format

**Webpage:**
```
Author, A. A. (Year, Month Day). Title of webpage. Site Name. URL

Example:
Smith, J. D. (2024, January 15). Understanding modern cyber threats. CyberSecurity Today.
    https://cybersecuritytoday.com/modern-threats
```

**Social Media Post:**
```
Author, A. A. [@username]. (Year, Month Day). Content of the post up to first 20 words [Description of 
    audiovisuals]. Site Name. URL

Example:
Musk, E. [@elonmusk]. (2024, January 10). Excited to announce our new initiative [Thumbnail with logo].
    X. https://x.com/elonmusk/status/1234567890
```

**Company Profile (LinkedIn):**
```
Company Name. (n.d.). Home [Company page]. LinkedIn. Retrieved Month Day, Year, from URL

Example:
Acme Corporation. (n.d.). Home [Company page]. LinkedIn. Retrieved January 20, 2024, from
    https://linkedin.com/company/acme-corporation
```

**WHOIS Record:**
```
Registrar. (Year, Month Day). WHOIS record for domain.com. Retrieved from URL

Example:
GoDaddy. (2024, January 18). WHOIS record for example.com. Retrieved from
    https://whois.godaddy.com/en/whois-example.com
```

**Court Document:**
```
Name v. Name, Volume Source Page (Court Year). URL

Example:
Smith v. Jones, 123 F.Supp.3d 456 (S.D.N.Y. 2024). https://courtlistener.com/opinion/12345/
```

**News Article:**
```
Author, A. A. (Year, Month Day). Title of article. Publication Name. URL

Example:
Anderson, K. L. (2024, January 22). Major data breach exposes millions. The Cyber Times.
    https://cybertimes.com/breach-2024
```

**Database Record:**
```
Author or Organization. (Year). Title of record [Description]. Database Name. URL

Example:
Secretary of State, Delaware. (2024). Acme Corporation business registration [Corporate record].
    Delaware Business Entity Search. https://icis.corp.delaware.gov/Ecorp/EntitySearch/
```

### OSINT-Specific Examples

**Archive.org Snapshot:**
```
Author or Site Name. (Year, Month Day). Title of page [Web page snapshot]. Wayback Machine.
    URL of archived page (Original URL)

Example:
Tech News Daily. (2023, June 15). Breaking: Major merger announced [Web page snapshot]. Wayback Machine.
    https://web.archive.org/web/20230615120000*/technewsdaily.com/merger-news
    (Original: https://technewsdaily.com/merger-news)
```

**Pastebin/Dump Site:**
```
Author [Username if known]. (Year, Month Day). Title or first 20 words [Data dump]. Site Name. URL

Example:
Anonymous. (2024, January 5). Internal documents from XYZ Corp [Data dump]. Pastebin.
    https://pastebin.com/abc123xyz
```

## 2. MLA Format Guide (9th Edition)

### In-Text Citations

**Basic Format:**
```
(Author Page)               # e.g., (Smith 45)
Author stated "..." (Page). # e.g., Smith stated "..." (45).
```

**No Author (Use title):**
```
("Title" Page)              # Use first few words
("Cyber Threats" 12)
```

**Multiple Works, Same Author:**
```
(Smith, "First Title" 45)
(Smith, "Second Title" 78)
```

### Works Cited Format

**Webpage:**
```
Author Last Name, First Name. "Title of Webpage." Website Name, Publisher (if different), 
    Date of publication, URL.

Example:
Smith, John D. "Understanding Modern Cyber Threats." CyberSecurity Today, 15 Jan. 2024,
    cybersecuritytoday.com/modern-threats.
```

**Social Media Post:**
```
Author [@username]. "Text of post." Social Media Platform, Date of post, Time of post, URL.

Example:
Musk, Elon [@elonmusk]. "Excited to announce our new initiative." X, 10 Jan. 2024, 14:32,
    x.com/elonmusk/status/1234567890.
```

**LinkedIn Profile:**
```
Last Name, First Name. LinkedIn profile. LinkedIn, URL. Accessed Day Month Year.

Example:
Smith, John. LinkedIn profile. LinkedIn, linkedin.com/in/johnsmith. Accessed 20 Jan. 2024.
```

**WHOIS Lookup:**
```
"WHOIS Record for [domain.com]." Registrar Name, Date accessed, URL.

Example:
"WHOIS Record for example.com." GoDaddy, 18 Jan. 2024, whois.godaddy.com/en/whois-example.com.
```

**Court Case:**
```
Name v. Name. Volume Source Page (Court Year). Database/Website, URL.

Example:
Smith v. Jones. 123 F.Supp.3d 456 (S.D.N.Y. 2024). CourtListener, courtlistener.com/opinion/12345/.
```

**News Article:**
```
Author Last Name, First Name. "Title of Article." Publication Name, Date of publication, URL.

Example:
Anderson, Karen L. "Major Data Breach Exposes Millions." The Cyber Times, 22 Jan. 2024,
    cybertimes.com/breach-2024.
```

**Public Record:**
```
Organization. Title of Record. Date of record. Database/Repository, URL. Accessed Day Month Year.

Example:
Secretary of State, Delaware. Acme Corporation Business Registration. 2024. Delaware Business Entity
    Search, icis.corp.delaware.gov/Ecorp/EntitySearch/. Accessed 20 Jan. 2024.
```

### OSINT-Specific Examples

**Archived Webpage:**
```
Author or Site Name. "Title of Page." Original site name, original date. Wayback Machine,
    web.archive.org/web/[timestamp]/[URL].

Example:
Tech News Daily. "Breaking: Major Merger Announced." Tech News Daily, 15 June 2023. Wayback Machine,
    web.archive.org/web/20230615120000/technewsdaily.com/merger-news.
```

**GitHub Repository:**
```
Username. Repository Name. GitHub, Year, github.com/username/repository.

Example:
johnsmith. osint-tools. GitHub, 2024, github.com/johnsmith/osint-tools.
```

## 3. Chicago Format Guide (17th Edition)

### Notes and Bibliography

**Footnote (First Citation):**
```
1. Author First Name Last Name, "Title of Webpage" (Website Name, Date), URL.

Example:
1. John D. Smith, "Understanding Modern Cyber Threats" (CyberSecurity Today, January 15, 2024),
   https://cybersecuritytoday.com/modern-threats.
```

**Footnote (Shortened):**
```
2. Smith, "Understanding Modern Cyber Threats."
```

**Bibliography Entry:**
```
Last Name, First Name. "Title of Webpage." Website Name. Date. URL.

Example:
Smith, John D. "Understanding Modern Cyber Threats." CyberSecurity Today. January 15, 2024.
    https://cybersecuritytoday.com/modern-threats.
```

**Social Media:**
```
Footnote:
1. Elon Musk (@elonmusk), "Excited to announce our new initiative," X, January 10, 2024,
   https://x.com/elonmusk/status/1234567890.

Bibliography:
Musk, Elon (@elonmusk). "Excited to announce our new initiative." X. January 10, 2024.
    https://x.com/elonmusk/status/1234567890.
```

**LinkedIn:**
```
Footnote:
1. John Smith, LinkedIn profile, accessed January 20, 2024, https://linkedin.com/in/johnsmith.

Bibliography:
Smith, John. LinkedIn profile. Accessed January 20, 2024. https://linkedin.com/in/johnsmith.
```

**Court Case:**
```
Footnote:
1. Smith v. Jones, 123 F.Supp.3d 456 (S.D.N.Y. 2024).

Bibliography:
Smith v. Jones. 123 F.Supp.3d 456 (S.D.N.Y. 2024).
```

**Public Record:**
```
Footnote:
1. Delaware Secretary of State, "Acme Corporation Business Registration," corporate record,
   accessed January 20, 2024, https://icis.corp.delaware.gov/Ecorp/EntitySearch/.

Bibliography:
Delaware Secretary of State. "Acme Corporation Business Registration." Corporate record.
    Accessed January 20, 2024. https://icis.corp.delaware.gov/Ecorp/EntitySearch/.
```

### OSINT-Specific Examples

**Archived Page:**
```
Footnote:
1. Tech News Daily, "Breaking: Major Merger Announced," June 15, 2023, archived at Wayback Machine,
   https://web.archive.org/web/20230615120000/technewsdaily.com/merger-news.

Bibliography:
Tech News Daily. "Breaking: Major Merger Announced." June 15, 2023. Archived at Wayback Machine.
    https://web.archive.org/web/20230615120000/technewsdaily.com/merger-news.
```

**WHOIS Record:**
```
Footnote:
1. GoDaddy, "WHOIS Record for example.com," accessed January 18, 2024,
   https://whois.godaddy.com/en/whois-example.com.

Bibliography:
GoDaddy. "WHOIS Record for example.com." Accessed January 18, 2024.
    https://whois.godaddy.com/en/whois-example.com.
```

## 4. URL Citation Best Practices

### URL Formatting Rules

**Standard URL Format:**
```
https://example.com/path/to/resource
```

**With Query Parameters:**
```
https://example.com/search?q=osint&year=2024
```

**Archived URLs:**
```
https://web.archive.org/web/20240115120000*/example.com/page
```

### URL Shortener Policy

**Never cite shortened URLs directly.**

**Process:**
1. Expand the shortened URL
2. Document the expansion date
3. Cite the full URL
4. Note the original shortener in brackets

**Example:**
```
Full URL: https://example.com/very/long/path/to/resource
Originally shared as: bit.ly/3abc123 (expanded January 15, 2024)
```

### Deep Linking Best Practices

**Preferred:**
- Link directly to specific content when possible
- Include section anchors: `#section-name`
- Document the date accessed

**Example:**
```
https://example.com/report#findings-section (accessed January 20, 2024)
```

### URL Verification Checklist

Before citing any URL:
- [ ] URL resolves successfully (HTTP 200)
- [ ] Content matches what is being cited
- [ ] URL has been archived (Wayback Machine)
- [ ] Access date documented
- [ ] No personal tracking parameters removed (`?utm_source=...`)

### URL Redaction Guidelines

**When to Redact:**
- URLs containing personal information
- Internal/non-public URLs
- URLs with access tokens or credentials

**How to Redact:**
```
https://internal-company.example.com/resource [URL redacted for privacy]
```

**Alternative:**
```
Internal company database [URL not publicly accessible, accessed January 20, 2024]
```

## 5. Source Attribution Templates

### Template 1: Webpage

```
Author. "Title." Website, Date. URL (accessed Date).

Full Example (APA):
Smith, J. D. (2024, January 15). Understanding modern cyber threats. CyberSecurity Today.
    https://cybersecuritytoday.com/modern-threats (accessed January 20, 2024).
```

### Template 2: Social Media Profile

```
Author [@username]. Platform Profile. Platform, URL (accessed Date).

Full Example (APA):
Doe, J. [@johndoe]. X Profile. X, https://x.com/johndoe (accessed January 20, 2024).
```

### Template 3: Social Media Post

```
Author [@username]. "Content (first 20 words)..." Platform, Date, Time. URL.

Full Example (APA):
Doe, J. [@johndoe]. (2024, January 15). "Excited to share our latest research on cybersecurity
    trends..." [Image attached]. X. https://x.com/johndoe/status/1234567890
```

### Template 4: Database Record

```
Organization. "Record Title" [Type]. Database, Date. URL (accessed Date).

Full Example (APA):
Delaware Secretary of State. (2024). "Acme Corporation Business Registration" [Corporate record].
    Delaware Business Entity Search. https://icis.corp.delaware.gov/Ecorp/EntitySearch/
    (accessed January 20, 2024).
```

### Template 5: Archived Content

```
Author. "Title." Original Site, Date. Archived at Wayback Machine, Archive URL (accessed Date).

Full Example (APA):
Tech News Daily. (2023, June 15). "Breaking: Major merger announced." Wayback Machine.
    https://web.archive.org/web/20230615120000/technewsdaily.com/merger-news
    (Original: https://technewsdaily.com/merger-news, accessed January 20, 2024).
```

### Template 6: Leaked/Exposed Data

```
Author [if known] or Anonymous. "Description" [Data type, source if known]. Site/Repository, Date. URL.

Full Example (APA):
Anonymous. (2024, January 5). "XYZ Corp internal documents" [Leaked documents]. Pastebin.
    https://pastebin.com/abc123xyz

Note: Data verification required. Ethical considerations apply.
```

### Template 7: IP/Network Information

```
Service. "Lookup Result for [IP/Domain]." Service Name, Date. URL.

Full Example (APA):
Shodan. (2024, January 20). "Search results for 203.0.113.45." Shodan.
    https://www.shodan.io/host/203.0.113.45
```

### Template 8: Geolocation Data

```
Service. "Location Data for [Query]." Service, Date. Coordinates: Lat, Long. URL.

Full Example (APA):
Google Maps. (2024, January 20). "1600 Amphitheatre Parkway, Mountain View, CA."
    Coordinates: 37.4220, -122.0841. Google Maps.
    https://maps.google.com/?q=37.4220,-122.0841
```

## 6. Quick Reference Comparison

### Side-by-Side Comparison

| Element | APA 7th | MLA 9th | Chicago 17th |
|---------|---------|---------|--------------|
| In-text | (Smith, 2024) | (Smith 45) | Footnote |
| Author format | Last, F. M. | Last, First M. | Last, First M. |
| Date position | After author | At end | Varies |
| URL | Include | Include | Include |
| "Retrieved from" | Optional | Use "Accessed" | Use "Accessed" |
| Page numbers | Use for direct quotes | Always use | Optional |

### Common OSINT Source Quick Formats

**LinkedIn Profile:**
```
APA:   Smith, J. (n.d.). LinkedIn profile. LinkedIn. Retrieved January 20, 2024, from https://linkedin.com/in/jsmith
MLA:   Smith, John. LinkedIn profile. LinkedIn, linkedin.com/in/jsmith. Accessed 20 Jan. 2024.
Chicago: Smith, John. LinkedIn profile. Accessed January 20, 2024. https://linkedin.com/in/jsmith.
```

**Tweet/X Post:**
```
APA:   Smith, J. [@jsmith]. (2024, January 15). "Content..." [Image]. X. https://x.com/jsmith/status/123
MLA:   Smith, John [@jsmith]. "Content..." X, 15 Jan. 2024, x.com/jsmith/status/123.
Chicago: Smith, John [@jsmith]. "Content..." X, January 15, 2024. https://x.com/jsmith/status/123.
```

**WHOIS Record:**
```
APA:   GoDaddy. (2024, January 18). WHOIS record for example.com. https://whois.godaddy.com/
MLA:   "WHOIS Record for example.com." GoDaddy, 18 Jan. 2024, whois.godaddy.com/...
Chicago: GoDaddy. "WHOIS Record for example.com." Accessed January 18, 2024. https://whois.godaddy.com/
```

**Archived Page:**
```
APA:   Site Name. (2023, June 15). Page title. Wayback Machine. https://web.archive.org/...
       (Original: https://...)
MLA:   Site Name. "Page Title." Date. Wayback Machine, web.archive.org/web/...
Chicago: Site Name. "Page Title." Date. Archived at Wayback Machine. https://web.archive.org/...
```

## 7. Special Considerations for OSINT

### Ephemeral Content

**For content that may disappear:**
- Always archive before citing
- Include archive URL and original URL
- Note archive date separately from access date

**Example (APA):**
```
Smith, J. [@jsmith]. (2024, January 15). "Content..." X. Archived at https://web.archive.org/...
on January 15, 2024. https://x.com/jsmith/status/123 (Original deleted by author).
```

### Anonymized Sources

**When source identity is protected:**
```
Anonymous. (2024, January 15). "Forum post title." Dark Web Forum [Name redacted].
    https://example.onion/thread/123 [Access restricted for safety].
```

### Chain of Evidence

**For sources derived from other sources:**
```
Original: Smith, J. (2024). Primary research.
Cited via: Jones, A. (2024). Secondary analysis citing Smith. "Quote from Smith..."
    (Smith, 2024, as cited in Jones, 2024, p. 45).
```

### Multiple Versions

**When content has been updated:**
```
Smith, J. (2024, January 15; last modified January 20, 2024). "Title." Site.
    https://example.com/page [Version consulted: January 20, 2024].
```

### Cross-Platform Content

**Same content across multiple platforms:**
```
Primary: Smith, J. [@jsmith]. (2024, January 15). "Content..." [Original post]. X.
    https://x.com/jsmith/status/123

Mirrors: Also posted to: LinkedIn (https://...), Facebook (https://...)
```
